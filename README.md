# Seam & Sole Premium Store Web Application

A high-performance, responsive e-commerce storefront for graphic tees, sneakers, and curated lookbooks. The application is built using Next.js 14, React, Tailwind CSS, TypeScript, and Three.js (via React Three Fiber) for interactive 3D elements.

## Technical Architecture & Stack

### Frontend & Rendering
*   **Framework:** Next.js 14 (utilizing the App Router structure for optimized routing and layout management).
*   **Language:** TypeScript (strictly typed configurations for data structures, product catalogs, and component props).
*   **Styling:** Tailwind CSS (utility-first design patterns, custom spacing scales, fluid typography, and glassmorphism styling).
*   **3D Interactive Elements:** Three.js, React Three Fiber (R3F), and `@react-three/drei` for rendering a high-performance 3D orbit viewer for featured product models.
*   **Animations:** Framer Motion (orchestrating smooth micro-interactions, fade-in transitions, and slide-overs).

### State Management & Interactions
*   **Global Cart & Wishlist:** Client-side React Context providers managing transient state (adding, removing, quantity updating, and local persistence).
*   **Recommendation Quiz Flow:** Modular state machine evaluating user preferences to filter and suggest matched products from the catalog.

## Database & Content Strategy

### Dynamic Product Generator
The product catalog is programmatically defined and managed through a Node.js script located at `src/data/generate-products.js`. Running this script generates the static database at `src/data/products.json`.

*   **Total Catalog Size:** 110 real-world products consisting of:
    *   40 Sneakers (Nike, Adidas, Jordan, New Balance, Converse, Vans, Puma, ASICS, Reebok, Fila).
    *   40 Graphic & Basic Tees (Nike ACG, Supreme, Stussy, FOG Essentials, Palace, Patagonia, CDG, Carhartt WIP, Ralph Lauren, Kith).
    *   15 Hoodies & Crewnecks (Champion Reverse Weave, Stone Island, Off-White, Carhartt WIP).
    *   15 Accessories (Carhartt Canvas Totes, Supreme Nylon Bags, G-Shock Watches, caps, beanies, socks).

### Advanced CDN Query Parameterizations for Hover States
To achieve seamless e-commerce image transitions on product hover without asset overhead or 404 broken image states, the generator applies dynamic query operations using the Unsplash Imgix CDN:
*   **Sneakers & Accessories:** Hover images are generated by appending `&flip=h` to the primary image URL. This mirrors the product horizontally, showing it from a different angle while guaranteeing 100% brand, color, and design consistency.
*   **Apparel (Tees & Hoodies):** To prevent text or logo reversal from horizontal mirroring, the hover image is generated by appending `&crop=focalpoint&fp-z=1.4`. This provides a premium close-up focal zoom on the fabric detail and print quality.
*   **Special Case Overrides:** Specific key items, such as the Nike Air Force 1 '07 "White", utilize manually paired dual-angle photography (e.g., standard wood background for primary, dark flat-lay for hover) to enhance visual depth.

### Pricing Strategy
*   **Discount Operations:** The catalog applies a global 35% discount calculated relative to normal retail price.
*   **Sale badge rendering:** Approximately 30% of the products are marked with visual discounts (`oldPrice` set to standard retail, `price` set to 35% below retail). The rest list the discounted price directly.

## Directory Structure

```
├── public/                 # Static assets (3D GLTF models, Rive animations, Lottie configs)
├── src/
│   ├── app/                # Next.js 14 App Router routes (shop, product, lookbook, fit-studio, quiz)
│   ├── components/         # Reusable UI components (product cards, checkout, navigation, 3D orbits)
│   ├── data/               # Static JSON databases (products.json, lookbook.json, reviews.json)
│   ├── hooks/              # Custom React hooks (cart operations, wishlist, viewport dimensions)
│   ├── lib/                # Utility helpers, product filters, and formatters
│   ├── types/              # TypeScript interfaces and type declarations
│   └── scratch/            # Development diagnostic and testing scripts
```

## Compilation and Deployment

The storefront is fully pre-rendered during build using Static Site Generation (SSG) for routes like `/product/[slug]` and `/lookbook/[slug]`. 

### Installation
```bash
npm install
```

### Run Local Development Server
```bash
npm run dev
```

### Build Production Bundle
```bash
npm run build
```

The compiler runs type-safety checks and lint validations before exporting static pages.
